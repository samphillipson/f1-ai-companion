# Forgot Password Functionality Walkthrough

## Overview
We've successfully added a full "Forgot Password" flow to the application. Users can now securely reset their passwords via email verification.

## Changes Made
- **Database Schema Updated**: Added `resetPasswordToken` and `resetPasswordExpires` to the `User` model in `prisma/schema.prisma` to securely track password reset requests.
- **Email Integration**: Expanded `src/lib/mailer.ts` to include a `sendPasswordResetEmail` function that generates a professional, branded email containing the secure reset link.
- **Forgot Password Flow**: 
  - Created a new `src/app/forgot-password/page.tsx` UI allowing users to enter their email address.
  - Created `src/app/api/auth/forgot-password/route.ts` to securely generate a 1-hour expiration token, save it to the database, and trigger the email.
- **Reset Password Flow**:
  - Created `src/app/reset-password/page.tsx` UI that takes the secure token from the URL and allows the user to set a new password.
  - Created `src/app/api/auth/reset-password/route.ts` to validate the token's authenticity and expiration before hashing and saving the new password.
- **UI Navigation**: Added a "Forgot password?" link directly into the `src/app/login/page.tsx` form for easy access.

## Next Steps / Manual Verification
Before this can work, **you must push the new database fields to Supabase**.

1. Please run the following command in your terminal to update the database schema:
   ```bash
   npx dotenv-cli -e .env.local -- npx prisma db push
   ```
2. Once the database is updated, start your dev server (`npm run dev`).
3. Navigate to the Login page and click the new "Forgot password?" link.
4. Enter your registered test email address and click send.
5. Check your email for the reset link, click it, and try resetting your password!
