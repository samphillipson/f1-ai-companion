## Proposed Changes

### Refactoring Authentication for Server Use

#### [MODIFY] `route.ts` (NextAuth)
- Extract the NextAuth configuration into an exported `authOptions` object. This is necessary so we can securely check the user's session on the server side (e.g., in `page.tsx`) without needing client-side JavaScript.

### Creating the Account UI

#### [NEW] `src/app/account/page.tsx`
- A Server Component that will call `getServerSession()` to verify the user is logged in.
- If not logged in, it will redirect them to `/login`.
- If logged in, it will render the client-side account dashboard.

#### [NEW] `src/app/account/AccountClient.tsx`
- A sleek, modern Client Component using the F1 AI dark theme and glassmorphism.
- It will display the user's email address.
- It will contain a secure form to change the password, requiring both the `currentPassword` and `newPassword`.

### Creating the API Endpoint

#### [NEW] `src/app/api/user/password/route.ts`
- A new secure API endpoint designed to handle password updates.
- **Logic flow:**
  1. Verify the user is authenticated via NextAuth session.
  2. Retrieve the user from the PostgreSQL database.
  3. Compare the provided `currentPassword` against the stored hash using `bcrypt`.
  4. If valid, hash the `newPassword` and update the user record in the database using Prisma.
  5. Return a success response.

### Updating Navigation

#### [MODIFY] `src/app/page.tsx`
- Update the top navigation bar to check the user's session.
- If logged in: Show an "Account" button and a "Sign Out" button.
- If logged out: Keep the current "Login" button.
